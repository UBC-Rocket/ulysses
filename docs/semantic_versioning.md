## Semantic Versioning
We follow semantic versioning (MAJOR.MINOR.PATCH):

### Version Numbers
- **MAJOR**: Breaking changes, major architectural updates
- **MINOR**: New features, sensor support, backwards compatible
- **PATCH**: Bug fixes, minor improvements

### Release Tags
- **Stable releases**: `v1.2.3`
- **Release candidates**: `v1.2.3-rc.1`
- **Beta versions**: `v1.2.3-beta.1`
- **Development builds**: `v1.2.3-dev.20241215`

### Version Management
- Tag releases on `main` branch only
- Include changelog with each release
- Document hardware compatibility changes
- Maintain LTS versions for stable platforms
